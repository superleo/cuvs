# Phase 04 — muVS Packaging and Prefix Rename (TDD Record)

## Goal

Implement the build-time prefix rename tooling so that when
`CUVS_GPU_BACKEND=MUSA`, the project produces artifacts named `muvs`
(library, headers, namespace, package) instead of `cuvs`. Also deliver a
compatibility header that lets existing cuVS code compile against muVS
with no source changes.

This phase implements ADR-006 (separate package identity) and ADR-007
(build-time prefix rename) from the system design.

## Bounded Context: Integration

All artifacts in this phase belong to the **Integration Context** (CMake,
packaging, tooling).

## TDD Step 2: Green (implement minimal code to pass)

### 2a. `tools/musa/rename_cuvs_to_muvs.py`

Core design decisions:

- **Replacement table**: three case-sensitive rules applied via a single
  compiled regex: `cuVS→muVS`, `CUVS→MUVS`, `cuvs→muvs`.
- **No CUDA touching**: the regex only matches `cuvs`/`CUVS`/`cuVS` — it
  never matches `cuda`/`CUDA`/`cuDA` because those substrings don't appear
  in the pattern.
- **Text vs binary**: files with known text extensions are read, renamed,
  and written; binary files are copied unchanged.
- **`rename_content(text)`**: pure function for text replacement.
- **`rename_path(relpath)`**: pure function for path component replacement.
- **`rename_tree(src, dst)`**: walks a directory tree, applies both.
- **`generate_compat_header()`**: returns a string containing `#define`
  mappings from 25 cuVS C API symbols to their muVS equivalents.
- **CLI**: `--src`/`--dst` for tree rename, `--generate-compat-header` for
  the compat header.

### 2b. `c/include/muvs/compat/cuvs_compat.h`

Generated by running:
```bash
python tools/musa/rename_cuvs_to_muvs.py --generate-compat-header --dst c/include
```

Maps 25 symbols:
- Types: `cuvsError_t`, `cuvsResources_t`, `cuvsStream_t`, `cuvsLogLevel_t`
- Functions: `cuvsResourcesCreate`, `cuvsStreamSet`, `cuvsStreamGet`, etc.
- Enums: `CUVS_ERROR`, `CUVS_SUCCESS`

### 2c. `cpp/cmake/modules/ConfigureMusvPackaging.cmake`

Sets 6 naming variables based on the selected backend:

| Variable | CUDA | MUSA |
|---|---|---|
| `CUVS_OUTPUT_NAME` | `cuvs` | `muvs` |
| `CUVS_OUTPUT_NAME_C` | `cuvs_c` | `muvs_c` |
| `CUVS_EXPORT_SET` | `cuvs-exports` | `muvs-exports` |
| `CUVS_INSTALL_INCLUDE_DIR` | `include/cuvs` | `include/muvs` |
| `CUVS_NAMESPACE` | `cuvs` | `muvs` |
| `CUVS_PACKAGE_NAME` | `cuvs` | `muvs` |

Also provides:
- `cuvs_apply_muvs_rename(<target>)` — sets `OUTPUT_NAME` on a CMake target.
- Two custom targets (project mode only): `muvs_generate_renamed_headers`
  and `muvs_generate_compat_header` that invoke the Python script.

### 2d. Wiring into `ConfigureBackend.cmake`

Added `include(ConfigureMusvPackaging.cmake)` in `ConfigureBackend.cmake`
before the feature-gate helper, so packaging variables are available to all
downstream CMake files.

## TDD Step 3: Refactor

### Python script
- Consolidated the three replacement rules into a single compiled regex
  (`_CONTENT_RE`) with a dictionary lookup for O(1) dispatch.
- `rename_tree` returns a list of output files for programmatic inspection.

### CMake module
- Guarded `add_custom_target` calls behind `CMAKE_PROJECT_NAME` check so
  the module works in both project mode and script mode (`cmake -P`).
- Used `CACHE INTERNAL` for all naming variables to ensure visibility
  across `add_subdirectory` boundaries.
